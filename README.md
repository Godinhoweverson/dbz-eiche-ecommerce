## Models
User data is stored in a relational database. The product model provides detailed information about each product, while the category model separates products into distinct categories, aiding in organization and user navigation.
For user interaction and feedback, the comments model allows users to leave comments on products, sharing their experiences and insights.
To complete the purchasing process seamlessly, the order and orderItem models handle all aspects, from saving shipping and payment information to recording product details for each order placed by the user.

- Product
Represents an item available for purchase in the shop. Products have various attributes such as name, description, price, and optional fields like color and image.

- Category
Represents a classification under which products are grouped. Categories help organize products, making it easier for users to browse and search for items based on their interests.

- Order
Represents a transaction made by a user to purchase one or more products. This model contains details such as user information, shipping address, order status, and payment-related data.

- OrderItem
Represents an individual product within an order. Each order may contain multiple order items, each specifying the product, quantity, and price associated with it.

- Comment
Represents a user's feedback or review on a purchased product. Users can submit comments to share their experiences or provide additional information about a product.

## Error Pages (403, 404, 500)

Custom error pages have been seamlessly integrated to ensure the site maintains its consistent look and feel across various scenarios:

Page Not Found (404): When a user attempts to access a page that doesn't exist, they are directed to a custom error page. This maintains coherence with the site's design while informing users about the missing content.

Forbidden (403): If a user tries to access a page without the necessary permissions, they are presented with a custom error page. This reinforces site security measures while guiding users appropriately.

Server Error (500): In the event of a server error, users are directed to a custom error page, ensuring a smooth transition and providing reassurance during unexpected technical difficulties.

Each error page includes a convenient link for users to easily return to the Main page. This intuitive feature enhances user experience by offering a seamless and user-friendly navigation path, even in the face of unexpected errors.

## Deployment

The deployment process entails four essential steps:

#### Backend Database Setup (PostgreSQL)

- Create an ElephantSQL account if necessary and log in.
- Generate a new instance, avoiding special characters in the name.
- Choose the appropriate plan and region.
- Copy the URL from the dashboard into your project's env.py template for the DATABASE_URL variable.
- Update settings.py in your Django project accordingly.
- Confirm connectivity by migrating database structures with manage.py commands.

#### Static Files Hosting Setup (AWS)

- Create an AWS account if needed and log in to S3.
- Establish a bucket with appropriate ACLs enabled.
- Configure bucket settings for static website hosting, adapting permissions and policies.
- Create a new IAM user group and user, attaching necessary permissions policies.
- Retrieve access keys from the generated .csv file.
- Implement serving static files using AWS in the Django app, with environment variable configuration.

##### Heroku Application Setup

- Log in to Heroku and create a new app, specifying the region.
- Connect the app to your GitHub repository.
- Obtain the SECRET_KEY and DATABASE_URL from respective sources.
- Copy AWS access keys from the .csv file.
- Set environment variables in Heroku's Config Vars, including secret keys and URLs.

#### Stripe Integration Setup

- Create a Stripe account if required and access the Developer tab.
- Copy both STRIPE_PUBLIC_KEY and STRIPE_SECRET_KEY from API Keys into environment variables.
- Optionally, add STRIPE_WH_SECRET for webhook testing.
- Securely store and extract keys in settings.py using environment variables to avoid hard-coding.
